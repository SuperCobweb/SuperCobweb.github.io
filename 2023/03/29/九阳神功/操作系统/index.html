操作系统

进程

进程和线程的区别

  • 根本：进程是操作系统资源分配的基本单位，线程是处理器任务调度和执行的基本单位
  • 切换资源开销：每一个进程都有自己的独立的代码和数据空间，程序之间切换会有较大的开销。线程是轻量的进程，同一个进程的线程共享代码和数据空间，每一个线程都有自己的运行栈和程序计数器，线程之间切换代价小
  • 包含关系：如果一个进程有多个线程，则执行过程不是一条线的，而是多个线程共同完成的。线程是进程的一部分，所以线程也被称为轻量级进程
  • 内存分配：同一进程的线程之间共享资源和地址空间，进程之间是独立的
  • 影响关系：一个进程崩溃之后，在保护模式下不会对其他进程产生影响。但是一个进程的线程崩溃，整个进程就会死掉，所以多进程比多线程健壮。
  • 执行过程：每个进程都拥有自己的程序入口、执行顺序、程序出口，但是线程不能独立运行，必须依托于进程，由应用程序提供多个线程的执行控制，两者均可并发执行。

协程

协程是微线程，在子程序内部执行，可在子程序内部中断，转而执行别的子程序，在适当的时候再返回执行。

协程与线程的区别：

  • 协程执行效率高，协程直接操作栈，没有内核切换的开销，协程切换开销比线程小
  • 协程不需要多线程锁的机制，因为多个协程从属于一个线程
  • 一个线程可以有多个协程

进程切换比线程切换更消耗资源

进程切换的时候需要刷新TLB并获取新的地址空间，然后切换硬件上下文和内核栈；线程只需要切换硬件上下文和内核栈。

进程间通信

进程通信：进程之间传输信息

进程同步：控制多个进程按照一定顺序执行

进程间通信的几种方式：管道、命名管道、消息队列、信号量、共享存储、Socket

  1. 管道

    管道通过pipe函数创建的，fd[0]用于读， fd[1]用于写

    具有一下限制：

    • 只支持半双工通信
    • 只能在父子进程或兄弟进程中使用

  2. 命名管道

    去除了匿名管道只能在父子进程中使用的限制。

  3. 消息队列

    消息队列是存放在内核中的消息链表，匿名管道存放在内存中的文件中、命名管道存放在实际的磁盘介质中。

    消息队列允许一个或多个进程向他写入或读取消息

    消息队列可以实现消息的随机查询，不一定要以先进先出的次序读取

  4. 信号量

    他是一个计数器，用于解决进程和线程并发执行时的同步问题。

    对信号量的操作分为P操作和V操作，P操作是将信号量的值减一，V操作是将信号量的值加一。当信号量的值小于等于0之后再执行P操作，当前线程就会被阻塞直到线程执行了V操作后将信号量的值增加到大于0时。

  5. 共享存储

    允许多个进程共享一个给定的存储区，需要使用信号量用来同步对共享存储的访问。

    采用共享内存通信的一个显而易见的好处是效率高，因为进程可以直接读写内存，而不需要任何数据的拷贝。对于像管道和消息队列等通信方式，则需要在内核和用户空间进行四次的数据拷贝，而共享内存则只拷贝两次数据：一次从输入文件到共享内存区，另一次从共享内存区到输出文件。

  6. 套接字

    与其他通信机制不同，它可用于不同机器之间进程通信

  7. 信号

    信号是进程间通信中唯一的异步通信机制，通知接受信号的进程有哪些事情发生了。

死锁

死锁是指连个或两个以上的进程在执行过程中，由于资源竞争或是由于彼此通信而造成阻塞的一种现象，若无外力作用，他们都将无法推进下去。

产生死锁的条件

  1. 互斥条件

    某个资源只能被一个进程所占有，不能同时被两个或多个进程占有

  2. 不可剥夺条件

    分配给一个进程的资源不可强制地被抢占，它只能被占有它的进程显示释放

  3. 占有和等待条件

    进程在请求资源得不到满足的情况下，不释放占有的资源

  4. 循环等待条件

    存在循环等待链

解决死锁

鸵鸟策略

不进行处理。因为解决死锁问题的代价很高，因此鸵鸟策略这种不采取任务措施的方案会获得更高的性能。

防止死锁（预防死锁）

至少破坏死锁产生的四个条件

  1. 破坏互斥条件

    使资源同时访问而非互斥访问

  2. 破坏不可剥夺条件

    占有资源的进程要申请新资源，必须主动释放所有的已占资源

  3. 破坏占有和等待条件

    一种实现方式：所有进程在开始执行前请求需要所有的资源

  4. 破坏循环等待条件

    给资源统一编号，所有进程必须按照资源标号依次进行请求资源

避免死锁

死锁防止方法能够防止死锁发生，但是会降低系统并发度，因此应该避免死锁。

系统在进行资源分配前预先计算资源分配的安全性，若此次分配不会导致系统进入不安全状态，则将资源分配给进程，否则，进程等待，其中最具有代表性的算法是银行家算法。

银行家算法：首先需要定义状态和安全转态。转态包含两个向量：resouce（系统每种资源的数量）、available（未分配给进程的每种资源的数量）两个矩阵 claim（进程对资源的需求）和 allocaiton（当前分配给进程的资源）。安全状态：至少有一种资源分配序列不会导致死锁。当进程请求一组资源时，假设同意该请求会改变系统的状态，然后判断是否处于安全状态，如果是，统一这个请求，如果不是，阻塞该进程直到同意该请求后系统处于安全状态。

死锁检测与死锁恢复

检测死锁：

  1. 首先为每个进程和资源指定一个唯一的标号
  2. 然后建立资源分配表和进程等待表
  3. 查看进程等待图中是否出现环
  4. 若出现环，表明有死锁

解除死锁：

  1. 资源剥夺法

    将一些死锁的进程挂起来，并且抢占他们的资源，分配给其他死锁的进程

  2. 撤销进程法

    强制撤销部分死锁的进程并剥夺资源。撤销的原则按照进程的优先级和撤销进程的代价高低进行

  3. 进程回退法

    让一个或多个进程回退到足以避免死锁的地步，进程回退时是资源释放资源而非剥夺。

自旋锁

当一个线程去获取锁的时候，这个锁被其他线程获取，那么这个线程就无法得到这把锁，该线程就会等待下去，每个一段时间尝试获取，这种循环加锁的机制叫做自旋锁。

自旋锁存在的问题：

  • 如果某个锁占有的时间较长，会导致其他等待线程循环等待，消耗CPU，使cpu使用率极高
  • 这是一个不公平锁，无法满足等待时间最长的线程获取锁，会出现线程饥饿问题

内存

虚拟内存

是为了让物理内存扩充成更大的逻辑内存，从而让程序获得更多的可用内存。

页面置换算法

  1. 最优页面置换算法 OPT

    被换出去的页面将是最长时间内不在被访问的，通常可以保证最低的缺页率，理论上的算法，因为无法知道一个页面多长时间不再被访问。

  2. 最近最久未使用算法 LRU

    LRU将最近最久未使用的页面换出。为了实现LRU，需要再在内存中维护一个所有页面的链表，当一个页面被使用的时候，将这个页面移到链表的表头，这样就能保证表尾的页面是最久未使用的。每次访问都要更新LRU链表，代价高。

  3. 最不经常使用算法 LFU

    当一个缺页发生的时候，选择访问次数最少得那个页面，淘汰之。

    对每个页面设置一个访问计数器，每当一个页面被访问时，计数器加一，当缺页发生时，淘汰计数器最小的那个页面。

  4. 先进先出 FIFU

    选择换出的页面是最先使用的页面

  5. 第二次机会算法

    FIFU会将会将经常使用的页面置换出去，为了避免这个问题，对该算法做了修改：

    当页面被访问时设置该页面的R位（使用位）为1，需要替换的时候检查最老页面的R位，如果R位为0，那么这个页面既老又没有被使用，可以立刻替换掉；如果是1，就将R位清0，并把该页面放到链表的尾端，修改他的装入时间，使它就像刚装入一样，然后继续从链表头部开始搜索。

  6. 时钟算法

    第二次机会算法需要再链表中移动页面，降低了效率，时钟算法使用环形链表将页面连接起来，再使用一个指针指向最老的页面。
